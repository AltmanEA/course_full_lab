# Next.js 15: Изоляция loading.tsx через Route Groups

## Задача
В приложении есть проблема с отображением loading state. Файл `loading.tsx` показывается на всех страницах (`/`, `/invoices`, `/customers`), хотя должен отображаться только на главной странице `/`.

## Инструкция
1. Изучите текущую структуру папок в `app/`
2. Определите, как ограничить область действия `loading.tsx` только для главной страницы
3. Используйте Route Groups для изоляции loading state

## Требования
- На `/` должен отображаться loading indicator
- На `/invoices` контент должен загружаться сразу без loading indicator
- На `/customers` контент должен загружаться сразу без loading indicator

## Подсказка
Посмотрите документацию Next.js про папки с именем в скобках `(name)`.
