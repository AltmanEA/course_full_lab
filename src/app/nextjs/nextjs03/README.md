# Next.js 03: Lazy Loading Images

## Задача
Добавить ленивую загрузку (lazy loading) для изображений на странице, чтобы улучшить производительность загрузки и пользовательский опыт.

## Проблема
В исходном состоянии все изображения загружаются сразу при открытии страницы, что приводит к:
- Медленной начальной загрузке страницы (увеличенный LCP)
- Неэффективному использованию bandwidth
- Плохому пользовательскому опыту на медленных соединениях
- Увеличенному потреблению данных

## Что нужно сделать
1. Настроить lazy loading для изображений, которые не видны в viewport
2. Добавить placeholder'ы для изображений во время загрузки
3. Настроить правильные размеры изображений для разных устройств
4. Проверить, что критические изображения загружаются сразу
5. Убедиться, что lazy loading не влияет на SEO

## Концепция Performance Optimization
Lazy loading изображений помогает:
- Улучшить Core Web Vitals (особенно LCP - Largest Contentful Paint)
- Сократить время начальной загрузки страницы
- Экономить bandwidth пользователей
- Улучшить пользовательский опыт
- next/image реализует smart lazy loading автоматически
