# Next.js 05: Dashboard Nested Layout

## Задача
Создать общий вложенный layout для dashboard страниц, чтобы избежать дублирования кода навигации и структуры.

## Проблема
В исходном состоянии каждая dashboard страница дублирует:
- Код навигации и сайдбара
- Общую структуру layout'а
- Стили и компоненты интерфейса
Это приводит к:
- Дублированию кода и сложности поддержки
- Непоследовательности в UI
- Увеличенному размеру бандла
- Сложности внесения изменений в навигацию

## Что нужно сделать
1. Создать вложенный layout для dashboard сегмента
2. Перенести общую навигацию и структуру в layout
3. Убрать дублирование кода из отдельных страниц
4. Убедиться, что все dashboard страницы используют общий layout
5. Проверить, что навигация работает корректно

## Концепция Nested Layouts
Вложенные layout'ы в Next.js позволяют:
- Создавать иерархическую структуру страниц
- Избегать дублирования кода
- Обеспечивать консистентный UI
- Улучшать maintainability кода
- Создавать переиспользуемые компоненты интерфейса
