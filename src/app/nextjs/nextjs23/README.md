# Next.js 23: Server Actions - Обработка ошибок с try/catch

## Задача
Добавить обработку ошибок в Server Action createInvoice с использованием try/catch. При ошибке пользователь должен видеть информативное сообщение вместо страницы сбоя.

## Инструкция
1. Найдите TODO комментарий в файле `actions.ts`
2. Оберните вызов `createInvoiceData` в блок try/catch
3. В блоке catch верните объект ошибки: `{ error: "сообщение ошибки" }`
4. Добавьте `revalidatePath` после успешного выполнения
5. Вызовите `redirect` после revalidatePath
6. В файле `page.tsx` используйте хук `useActionState` для обработки ошибок

## Требования
- Обернуть код мутации в try/catch
- Использовать `return { error: ... }` для возврата сообщения об ошибке
- Добавить `revalidatePath` для обновления данных
- Вызвать `redirect` после успешного выполнения
- Использовать `useActionState` для отображения ошибок пользователю

## Подсказка
Функция `createInvoiceData` выбрасывает ошибку при сумме меньше 1000. После try/catch можно вызывать `redirect`, так как он работает через throw error и не выполняется в блоке catch. Хук `useActionState` позволяет получать состояние ошибки из Server Action.
