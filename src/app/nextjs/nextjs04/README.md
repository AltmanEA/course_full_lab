# Next.js 04: Creating Route Segments

## Задача
Создать правильную структуру папок для нового route segment. Страница должна быть доступна по пути /nextjs/nextjs04/demo, а не в корне.

## Проблема
В исходном состоянии page.tsx файл находится не в той папке, что создает неправильный URL маршрут. Это приводит к:
- Неправильному URL'у страницы (доступна в корне, а не по нужному пути)
- Нарушению структуры маршрутизации
- Сложности в навигации и понимании архитектуры приложения
- Проблемам с SEO и пользовательским опытом

## Что нужно сделать
1. Создать правильную структуру папок для route segment
2. Переместить page.tsx в нужную папку
3. Убедиться, что URL изменился на /nextjs/nextjs04/demo
4. Проверить, что навигация работает корректно
5. Обновить все ссылки на новый URL

## Концепция File-based Routing
Next.js использует file-based routing:
- URL структура отражает структуру папок в app directory
- Каждая папка создает новый route segment
- page.tsx в папке становится страницей для этого маршрута
- Правильная структура папок улучшает maintainability и понятность кода
