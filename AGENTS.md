Задачи для студентов изучающих технологии nextjs, tailwind, drizzle, trpc.

# Структура задачи.

```
src/app/{technology}/{task_name}/
├── page.tsx             # Страница с проблемой
├── README.md            # Описание задачи (минимум текста)
└── {task_name}.spec.ts  # Playwright тесты
```
Код задачи может быть в двух состояниях:
- ЗАДАНИЕ - код с проблемой, которую предлагается решить студентам
- РЕШЕНИЕ - код с правильным  решением.
Имена задач выбираются как  "{technology}01", "{technology}02" и т.д. 
Для сложных задач могут использоваться другие файлы и подпапки.
Для решения задачи нужно исправить или написать 3 до 7 строк кода.
Задача не должна решаться средствами vscode со стандартными плагинами, а требовать написание кода.  
Для решения задач должно быть достаточно знания только основных, часто используемых команд.
Задача должна проверятся тестами без анализа исходного кода.

# Работа над проектом.

- Пользователь ИИ (далее - просто пользователь) предлагает тему одной или нескольких задач.
- ИИ предлагает общее описание и структуру задач. В описании указывается, что должны сделать студенты и как это будет тестироваться.
- Пользователь согласует задачу, ИИ создает задачи по одной. Для каждой задачи:
    - создает РЕШЕНИЕ, проверяет его 
    - преобразует код в ЗАДАНИЕ, проверяет что тесты проваливаются
    - уточняет условие, при необходимости корректируя код предыдущих пунктов.
- Любые изменения в структуре или условиях задачи нужно согласовывать с пользователем.
- При ответе не делать предположений, если есть несколько вариантов решений - спросить пользователя.
- Если требуется выполнить сложные многошаговые действия - спросить подтверждение пользователя.

# Оформление задачи
В файле README.md должно быть краткое описание задачи, без подсказок в виде имен конкретных функций.
Пример README.md в файле /src/app/nextjs/nextjs01/README.md.
Включать TODO комментарии без конкретных ключевых слов.
Избегать сложных зависимостей между компонентами.
Страницы приложения, которые нужно редактировать при решении задачи, должны содержать минимум текста и стилей.

Тестирование задач: 
Используйте Playwright для E2E тестирования
Отладочный сервер запущен пользователем, нужный каталог выбран, для тестирование команда 'npx playwright test {имя_задачи}'.
Тесты не должны анализировать исходный код, а проверять результат работы.
Тесты должны проваливаться для состояния ЗАДАНИЕ
Тесты должны проходить для состояния РЕШЕНИЕ

# Технические детали
В терминале использовать команды windows (удалить - 'del', каталог - 'dir' и т.п.)
Не оставлять бэкапы решений.
Изображения и др. размещать в public/images/
Использовать локальные пути /images/filename.ext
Проект должен работать без интернета
